High-level plan (Gopher vs HTTP) — with % + rubrics

1. Definitions + scope (Gopher vs HTTP)
    - Supports: Presentation (20%) → Inquiry, Central Message, Organisation
    - Supports: Report (10%) → Context & Purpose, Language Use

2. Problem + research question (compare speed/overhead + capability under same conditions)
    - Supports: Presentation (20%) → Central Message, Inquiry
    - Supports: Report (10%) → Context & Purpose, Content Development

3. Methodology setup (same content on both servers; vary latency/loss/bandwidth)
    - Supports: Presentation Requirements (10%) → Methodology: relevant methods
    - Supports: Report (10%) → Content Development, Organisation

4. Metrics + evidence collection (TTFB, total time, bytes; repeated runs; logs/Wireshark)
    - Supports: Presentation Technical Details (10%) → technical quality + complexity
    - Supports: Presentation (20%) → Inquiry (use of evidence), Visual Aids
    - Supports: Report (10%) → Sources + Citation, Content Development

5. Results + interpretation (graphs + explain trade-offs, not just numbers)
    - Supports: Presentation Technical Details (10%) → insights + interpretation
    - Supports: Presentation (20%) → Central Message development, Organisation
    - Supports: Report (10%) → Content Development (arguments + conclusion), Language Use

6. Applications/demo (where Gopher better vs where HTTP, tied to measured metrics)
    - Supports: Presentation (20%) → Visual Aids (demo), Central Message relevance
    - Supports: Presentation Technical Details (10%) → quality of work + insights
    - Supports: Report (10%) → Content Development (recommendations)

7. Conclusion + limitations (“best for which goals”, limitations of experiment)
    - Supports: Presentation (20%) → Central Message (clear takeaway)
    - Supports: Report (10%) → conclusions + limitations requirement, Context & Purpose

8. Project management section (who did what, what went well, problems + fixes)
    - Supports: Report (10%) → Project Management Overview requirement

Quick grading summary (so the team keeps focus)
- Presentation total = 20%
    - Presentation Requirements = 10% (problem + methodology)
    - Presentation Technical Details = 10% (complexity + insights)
- Report = 10%
- Total project = 30%
